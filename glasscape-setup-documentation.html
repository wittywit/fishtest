Overview & Requirements

+

What This Guide Does

Creates a luxury web-based file upload server on your Radxa Zero 3W with custom Glasscape branding. Users can drag-and-drop files via a web browser on the local network.

Requirements

  • Radxa Zero 3W running Armbian minimal
  • SSH access to the device
  • WiFi connection established
  • Basic terminal knowledge
End Result: Access at http://glasscape03.local:8080 from any network device

Filebrowser Installation

+

Step 1: System Update & Directory Creation

Updates package list and creates dedicated upload directory.

# Update system packages sudo apt update # Create directory for uploaded files sudo mkdir -p /home/uploads sudo chown $USER:$USER /home/uploads

Step 2: Download Filebrowser ARM64

Downloads correct version for ARM-based Radxa device.

# Verify architecture (should show aarch64) uname -m # Download Filebrowser for ARM64 wget https://github.com/filebrowser/filebrowser/releases/download/v2.24.2/linux-arm64-filebrowser.tar.gz # Extract archive tar -xzf linux-arm64-filebrowser.tar.gz # Verify binary exists ls -la filebrowser

Step 3: System Installation

Installs binary system-wide and makes executable.

# Move to system location sudo mv filebrowser /usr/local/bin/ # Make executable sudo chmod +x /usr/local/bin/filebrowser # Verify installation filebrowser version

Initial Configuration

+

Step 1: Configuration Directory

Creates directory structure for database and config files.

# Create config directory mkdir -p ~/.config/filebrowser

Step 2: Database & Admin User

Initializes database and creates default admin account.

# Initialize database filebrowser config init --database ~/.config/filebrowser/database.db # Create admin user (username: admin, password: admin) filebrowser users add admin admin --database ~/.config/filebrowser/database.db

Step 3: Server Configuration

Configures server to listen on all interfaces, port 8080.

# Configure server settings filebrowser config set --address 0.0.0.0 --port 8080 --root /home/uploads --database ~/.config/filebrowser/database.db

Step 4: Systemd Service Creation

Creates system service for automatic startup.

# Create systemd service file sudo nano /etc/systemd/system/filebrowser.service

Paste this content into the file:

[Unit] Description=Filebrowser After=network.target [Service] ExecStart=/usr/local/bin/filebrowser --database /home/glasscape03/.config/filebrowser/database.db User=glasscape03 Restart=always [Install] WantedBy=multi-user.target

Save: Ctrl + X, then Y, then Enter

Step 5: Service Activation

Enables and starts the service.

# Reload systemd sudo systemctl daemon-reload # Enable auto-start sudo systemctl enable filebrowser # Start service sudo systemctl start filebrowser # Check status sudo systemctl status filebrowser

Auto Login Setup

+

Step 1: Auto Login Configuration

Sets up automatic login for glasscape03 user.

# Create override directory sudo mkdir -p /etc/systemd/system/getty@tty1.service.d # Create configuration file sudo nano /etc/systemd/system/getty@tty1.service.d/override.conf

Paste this content:

[Service] ExecStart= ExecStart=-/sbin/agetty --autologin glasscape03 --noclear %I $TERM

Step 2: Service Enable

Activates auto-login service.

# Enable service sudo systemctl enable getty@tty1.service

Step 3: System Reboot

Applies all changes.

# Reboot system sudo reboot
After Reboot: Access at http://glasscape03.local:8080
Login: admin / admin

Change Login Credentials

+

Method 1: Web Interface (Recommended)

Change credentials through the web interface after first login.

  • Login to Filebrowser with admin/admin
  • Go to Settings → User Management
  • Click on the admin user
  • Change username and/or password
  • Save changes

Method 2: Command Line (Username + Password)

Change both username and password via terminal commands.

# Stop the service sudo systemctl stop filebrowser # Remove old admin user filebrowser users rm admin --database ~/.config/filebrowser/database.db # Create new admin with custom credentials filebrowser users add yournewusername yournewpassword --database ~/.config/filebrowser/database.db # Give admin permissions filebrowser users update yournewusername --perm.admin --database ~/.config/filebrowser/database.db # Start service sudo systemctl start filebrowser

Replace yournewusername and yournewpassword with your desired credentials.

Method 3: Password Only

Keep username as admin, change password only.

# Stop service sudo systemctl stop filebrowser # Change admin password only filebrowser users update admin --password yournewpassword --database ~/.config/filebrowser/database.db # Start service sudo systemctl start filebrowser
Security: Always change default credentials after first login for security

Remote Access with Tailscale

+

What is Tailscale

Tailscale creates a secure mesh network allowing remote access without port forwarding. It doesn't replace local access - both work simultaneously.

  • Encrypted connections between devices
  • No configuration of routers/firewalls needed
  • Access from anywhere in the world
  • Local network access still works normally

Step 1: Install Tailscale

Downloads and installs Tailscale on your Radxa device.

# Download and install Tailscale curl -fsSL https://tailscale.com/install.sh | sh

Step 2: Start and Authenticate

Starts Tailscale and provides authentication URL.

# Start Tailscale (shows authentication URL) sudo tailscale up

Copy the URL from terminal and open in browser. Sign in with Google/Microsoft/GitHub account and authorize the device.

Step 3: Verify Connection

Check Tailscale status and get device IP.

# Check status sudo tailscale status # Get Tailscale IP (starts with 100.x.x.x) sudo tailscale ip -4

Step 4: Enable Auto-Start

Configures Tailscale to start automatically on boot.

# Enable auto-start sudo systemctl enable tailscaled

Step 5: Install on Client Devices

Install Tailscale on devices you want to connect from.

  • Download from https://tailscale.com/download
  • Install on laptop, phone, tablet, etc.
  • Sign in with the same account
Remote Access:
SSH: ssh glasscape03@100.x.x.x
Filebrowser: http://100.x.x.x:8080
Replace 100.x.x.x with your Tailscale IP

Security Hardening

+

SSH Security

Harden SSH access with key authentication and disabled password login.

# Generate SSH key pair (run on your laptop) ssh-keygen -t ed25519 -C "glasscape-access" # Copy public key to device ssh-copy-id glasscape03@glasscape03.local

Then disable password authentication:

# Edit SSH config sudo nano /etc/ssh/sshd_config # Add or modify these lines: PasswordAuthentication no PermitRootLogin no PubkeyAuthentication yes # Restart SSH sudo systemctl restart ssh

Firewall Setup

Configure UFW firewall to only allow necessary ports.

# Install and configure firewall sudo apt install ufw # Default deny incoming sudo ufw default deny incoming sudo ufw default allow outgoing # Allow SSH and Filebrowser locally sudo ufw allow from 192.168.0.0/16 to any port 22 sudo ufw allow from 192.168.0.0/16 to any port 8080 # Allow Tailscale subnet sudo ufw allow from 100.64.0.0/10 # Enable firewall sudo ufw enable

Tailscale Access Control Lists (ACLs)

Restrict which devices can access which services.

  • Go to https://login.tailscale.com/admin/acls
  • Configure device-specific access rules
  • Use tags to group devices
  • Implement zero-trust access

Example ACL to restrict access:

{ "tagOwners": { "tag:glasscape": ["your-email@domain.com"], "tag:personal": ["your-email@domain.com"] }, "acls": [ { "action": "accept", "src": ["tag:personal"], "dst": ["tag:glasscape:22,8080"] } ] }

Device Tagging

Tag your Glasscape device for access control.

# Tag the device during setup sudo tailscale up --advertise-tags=tag:glasscape

Additional Security Measures

  • Auto-updates: Enable automatic security updates
  • Fail2Ban: Install to block brute force attempts
  • Regular monitoring: Check logs periodically
  • Minimal services: Only run necessary services
# Enable auto-updates sudo apt install unattended-upgrades sudo dpkg-reconfigure unattended-upgrades # Install Fail2Ban sudo apt install fail2ban # Check active services sudo systemctl list-units --type=service --state=active
Security Checklist:
✓ Change default passwords
✓ Use SSH keys, disable password auth
✓ Configure firewall
✓ Set up Tailscale ACLs
✓ Enable auto-updates
✓ Monitor access logs

Silent Boot & Custom Splash

+

What This Does

Creates a completely silent boot experience with optional custom splash image. No boot messages, no login prompts - straight to auto-login or custom branding.

  • Hides all kernel and systemd messages
  • Removes login prompts and banners
  • Optional: Shows custom Glasscape logo during boot
  • Creates professional, appliance-like experience

Method 1: Basic Silent Boot

Hide all boot messages for clean startup.

# Edit kernel parameters sudo nano /boot/armbianEnv.txt # Add or modify this line: extraargs=quiet splash loglevel=0 rd.systemd.show_status=false rd.udev.log_level=0 plymouth.enable=0

Method 2: Silent Boot with Custom Splash Image

Display custom Glasscape logo during boot process.

# Install Plymouth for splash screens sudo apt install plymouth plymouth-themes # Create custom theme directory sudo mkdir -p /usr/share/plymouth/themes/glasscape

Create the theme configuration:

# Create theme file sudo nano /usr/share/plymouth/themes/glasscape/glasscape.plymouth

# Set as default theme sudo plymouth-set-default-theme glasscape # Update initramfs sudo update-initramfs -u # Enable Plymouth in kernel parameters sudo nano /boot/armbianEnv.txt # Modify extraargs to: extraargs=quiet splash loglevel=0 rd.systemd.show_status=false rd.udev.log_level=0

Hide Login Messages

Remove all login banners and prompts.

# Remove login banners sudo rm -f /etc/motd /etc/issue /etc/issue.net # Create empty files to prevent recreation sudo touch /etc/motd /etc/issue # Update getty service for silent login sudo nano /etc/systemd/system/getty@tty1.service.d/override.conf

Update with silent flags:

[Service] ExecStart= ExecStart=-/sbin/agetty --autologin glasscape03 --noclear --noissue --skip-login --quiet %I $TERM StandardOutput=null StandardError=null

Disable System Status Messages

Hide systemd startup messages completely.

# Create systemd configuration sudo mkdir -p /etc/systemd/system.conf.d sudo nano /etc/systemd/system.conf.d/no-status.conf

Add this content:

[Manager] ShowStatus=no

Enable Auto-Updates Silently

Configure automatic security updates without user interaction.

# Install unattended upgrades sudo apt install unattended-upgrades # Configure automatically echo 'Unattended-Upgrade::Automatic-Reboot "true";' | sudo tee -a /etc/apt/apt.conf.d/50unattended-upgrades echo 'Unattended-Upgrade::Automatic-Reboot-Time "03:00";' | sudo tee -a /etc/apt/apt.conf.d/50unattended-upgrades # Enable auto-updates echo 'APT::Periodic::Update-Package-Lists "1";' | sudo tee /etc/apt/apt.conf.d/20auto-upgrades echo 'APT::Periodic::Unattended-Upgrade "1";' | sudo tee -a /etc/apt/apt.conf.d/20auto-upgrades

Apply Changes

Reload configuration and reboot to test.

# Reload systemd configuration sudo systemctl daemon-reload # Test Plymouth theme (optional) sudo plymouthd; sudo plymouth --show-splash; sleep 5; sudo plymouth quit # Reboot to see silent boot sudo reboot
Result: Professional appliance-like boot experience with optional Glasscape branding, silent operation, and automatic maintenance

Alternative: Simple Black Screen

For minimal setup without custom graphics.

# Just use kernel parameters for black screen sudo nano /boot/armbianEnv.txt # Add this line: extraargs=quiet loglevel=0 systemd.show_status=false rd.udev.log_level=0 rd.systemd.show_status=false console=tty3 # Reboot sudo reboot

This creates a blank screen during boot with no messages.
